1. Qué es MonitorMaat

MonitorMaat es un panel multi‑tenant que despliega y administra servicios de OpenVPN + Uptime Kuma dentro de contenedores Docker, uno por cliente. Cada cliente tiene su propio tenant aislado, con su red VPN, sus usuarios VPN, su Uptime Kuma con monitores y su facturación.

Como admin podés crear y administrar todo el sistema. Los clientes solo ven lo suyo.

2. Dashboard /

Muestra métricas en vivo del host: CPU, RAM, disco, número de tenants, usuarios y estado de los contenedores. Se refresca cada pocos segundos automáticamente.

3. Tenants (servicios) /tenants

Cada tenant es un servicio independiente con su par OpenVPN + Kuma. Listado paginado (10/15/20/50/100), buscable por nombre, dueño o empresa.

Crear tenant /tenants/new

  • Nombre: solo minúsculas, números y guión. Empieza con letra. Ej: maat, cliente-1.
  • El sistema asigna automáticamente puerto VPN (1194, 1195…), puerto Kuma (3001, 3002…) y subred (100.64.X.0/24).
  • Al crear se levantan ambos contenedores y queda el tenant accesible.

Vista de un tenant /tenants/<nombre>

  • Resumen: estado de OpenVPN y Kuma, dominio asignado, IP pública, subred VPN.
  • Acciones: arrancar / detener / reiniciar contenedores; eliminar tenant (zona peligrosa).
  • Usuarios VPN: alta de usuarios OpenVPN con descarga de .ovpn, asignación de subredes y baja.
  • Kuma: link directo al panel Uptime Kuma del tenant (con dominio si está configurado, IP+puerto si no).

💡 Si activaste un dominio wildcard en Red y dominios, los Kuma de los tenants quedan accesibles vía tenant.tu-dominio.com con HTTPS automático.

4. Usuarios VPN dentro de un tenant

Un usuario VPN es una credencial que se conecta al OpenVPN del tenant. NO es un usuario del panel.

  • Crear: nombre + descarga del .ovpn listo para importar en cualquier cliente OpenVPN.
  • Redes: a cada usuario VPN podés asociarle una o varias subredes (ej: 192.168.1.0/24) que se ruteen por su túnel — útil para Mikrotik / sucursales.
  • Eliminar: revoca el certificado y borra las rutas. La conexión activa se cae.

5. Usuarios del panel (clientes) /users

Aquí gestionás las cuentas de tus clientes que entran al panel.

  • Crear usuario: username, email, password, empresa, cuota de tenants, plan opcional.
  • Editar: cambiar email, empresa, cuota.
  • Reset de contraseña: genera una nueva password aleatoria y la muestra una sola vez.
  • Desactivar: bloquea el login. Si el usuario está logueado en ese momento, lo bota del panel.
  • Eliminar: borra la cuenta. No borra sus tenants (los podés transferir o eliminar a mano antes).

Los admins NO requieren plan ni facturación.

6. Facturación /billing

Listado de clientes con su estado de pago. Columnas:

  • Pago: ✅ Pagado / ❌ Impago según paid_until y la hora de suspensión.
  • Valor a cobrar: precio del plan asignado al cliente.
  • Días para corte: cuántos días faltan hasta que se le suspenda la cuenta.
  • Acciones: Gestionar (entra al detalle) y Registrar pago (modal flotante con el monto del plan ya prellenado).

Detalle de cliente /billing/<user_id>

  • Estado del cliente: días restantes, plan vigente, total cobrado por moneda.
  • Asignar plan: hasta que el cliente no tenga plan asignado, NO puede crear tenants ni pagar con PayPhone.
  • Cortesía (ajustar días): número positivo extiende, negativo reduce. Si la resta cae antes de hoy, queda en hoy. No requiere confirmación: aplica directo.
  • Historial de pagos: paginado, con fecha, monto, días cubiertos, método y operador.

💡 La hora exacta a la que se suspenden las cuentas que vencen ese día se configura en Planes (zona horaria Ecuador, UTC‑5).

7. Planes de suscripción /settings/plans

  • Hora de suspensión: hora del día (HH:MM, Ecuador) a la que se cortan las cuentas vencidas.
  • Crear plan: nombre, descripción, precio, moneda (USD/EUR), días de cobertura, orden, activo sí/no.
  • Activo: si está marcado, el plan aparece en el selector de "Asignar plan al cliente". Si lo desactivás, queda guardado pero no se puede asignar a clientes nuevos. Los que ya lo tienen asignado siguen pagándolo.
  • Editar: tabla inline. Cambiá precio, días, orden, etc.
  • Borrar: si el plan tiene pagos asociados queda solo desactivado (no se borra para preservar histórico).

8. Red y dominios /settings/network

Acá configurás los dominios para HTTPS automático con Caddy + Let's Encrypt:

  • Dominio del panel: ej. panel.tudominio.com apunta al MonitorMaat.
  • Dominio wildcard de tenants: ej. *.tudominio.com. Cada tenant queda servido vía nombre-tenant.tudominio.com.
  • Email Let's Encrypt: para los certificados.
  • SSL automático: Caddy renueva los certs solo, no hay que tocar nada.

Para ver el Caddyfile generado, usá /settings/network/caddyfile.

⚠ El DNS wildcard debe apuntar a la IP pública del server antes de activar HTTPS, sino Let's Encrypt falla.

9. PayPhone /settings/payphone

Pasarela de pago para cobrar a los clientes con tarjeta. Configurá:

  • Token de la cuenta PayPhone.
  • Store ID del comercio.
  • URL pública del panel: necesaria para los callbacks de retorno.

Una vez configurado, los clientes con plan asignado ven el botón "Pagar con PayPhone" en su sección de suscripción. El IVA Ecuador (15%) se calcula automáticamente.

10. SMTP /settings/smtp

Correo saliente para resets de contraseña, avisos de pago y notificaciones críticas. Configurá host, puerto, usuario, password, remitente y TLS. Botón Probar envío manda un correo de prueba a la dirección que indiques.

11. Bot Telegram /settings/telegram

Bot global del sistema. Permite enviar alertas a los admins y a los clientes que vinculen su cuenta.

  • Token del bot (lo das de alta con @BotFather en Telegram).
  • Chat ID admin: el grupo o usuario que recibe alertas críticas.
  • Probar: envía un mensaje al chat configurado.

12. Logs / Auditoría /logs

Registro inmutable de cada acción significativa. Filtrable por:

  • Origen: Sistema / Admin / Clientes.
  • Categoría: auth, user, tenant, vpn_user, network, system, settings.
  • Búsqueda libre: por usuario, tenant, acción o detalle.

Listado paginado (10/15/20/50/100). La preferencia de tamaño se guarda en tu navegador.

13. Tu cuenta admin /change-password

Cambiar tu contraseña. Te recomendamos:

  • Usar una contraseña fuerte (12+ caracteres, sin diccionario).
  • Si compartís el panel con otro admin, creá una cuenta admin separada para él (no compartas credenciales).

14. Solución de problemas comunes

Cliente no puede crear tenants
Verificá: 1) tiene plan asignado, 2) está al día con el pago, 3) no superó su cuota de tenants. Todo en /billing/<id>.
Tenant aparece en estado "missing"
El contenedor no existe. Mirá la página del tenant y usá Arrancar. Si falla, revisá los logs del host.
Kuma del tenant no abre con HTTPS
Verificá que el wildcard DNS apunte al server y que Caddy esté corriendo. Mirá /settings/network/caddyfile.
PayPhone devuelve error después de pagar
Confirmá que la URL pública en PayPhone matchee EXACTAMENTE al dominio por donde entró el cliente (HTTPS si aplica). El callback necesita llegar de vuelta.
El cliente no recibe correo
Probá el envío en SMTP. Verificá SPF/DKIM si tu proveedor de correo los exige.
