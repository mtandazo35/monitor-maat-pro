⚙ Instalar Caddy en el VPS

Caddy es el reverse proxy que maneja SSL automático. Hay que instalarlo una sola vez:

📋 Comandos paso a paso

1. Conectate al VPS por SSH y crear el container Caddy (elegí UNO de los dos modos):

Opción A — Caddy con docker run (rápido):

mkdir -p /opt/kumavpn/caddy /opt/kumavpn/caddy-data /opt/kumavpn/caddy-config /opt/kumavpn/letsencrypt

docker run -d \
  --name caddy-monitormaat \
  --restart unless-stopped \
  --network kumavpn-admin_default \
  -p 80:80 -p 443:443 -p 443:443/udp \
  -v /opt/kumavpn/caddy/Caddyfile:/etc/caddy/Caddyfile \
  -v /opt/kumavpn/caddy-data:/data \
  -v /opt/kumavpn/caddy-config:/config \
  -v /opt/kumavpn/letsencrypt:/certs:ro \
  caddy:2-alpine

Opción B — Agregar al docker-compose existente:

Edita /opt/kumavpn-docker/docker-compose.yml y agregá un servicio caddy con el mismo network. Después docker compose up -d.

2. Volvé a esta pantalla y guardá los dominios.

El panel detecta el container Caddy y aplica el Caddyfile automáticamente.

3. Verificá:

docker logs -f caddy-monitormaat

Vas a ver los pedidos a Let's Encrypt y los certs emitidos.

📖 SSL por API (modo 3) — cómo funciona por dentro / hacerlo a mano

El panel automatiza esto con la imagen oficial certbot/dns-cloudflare (no tenés que hacerlo a mano). Por si querés replicarlo en otro servidor:

1. Instalar Certbot + plugin (Ubuntu/Debian):

sudo apt update
sudo apt install certbot python3-certbot-dns-cloudflare

2. Token scoped en Cloudflare (plantilla Edit zone DNS, tu zona en Zone Resources).

3. Guardar credenciales:

sudo mkdir -p /etc/letsencrypt
echo 'dns_cloudflare_api_token = TU_TOKEN' | sudo tee /etc/letsencrypt/cloudflare.ini
sudo chmod 600 /etc/letsencrypt/cloudflare.ini

4. Emitir el wildcard (Certbot crea el TXT de validación, obtiene el cert y borra el TXT solo):

sudo certbot certonly \
  --dns-cloudflare \
  --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
  -d 'midominio.com' \
  -d '*.midominio.com'

Certs en /etc/letsencrypt/live/midominio.com/fullchain.pem y privkey.pem. En este panel quedan bajo /opt/kumavpn/letsencrypt/ y Caddy los usa vía tls; la renovación corre sola cada ~60 días (DNS puede quedar en nube naranja).
