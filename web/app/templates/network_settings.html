🌐 Red y dominios

Dominio base para Uptime Kuma

Si configurás un dominio aquí, los Uptime Kuma de cada tenant se mostrarán como <tenant>.tudominio.com en lugar de http://IP:puerto. Las VPN siguen mostrando la IP (necesario para OpenVPN endpoints).

Requiere: registro DNS wildcard *.tudominio.com → {{ public_ip or 'IP del VPS' }} y un reverse proxy (Caddy recomendado, abajo te genero el archivo).

Marcado por default. Caddy genera certs Let's Encrypt automáticos por subdominio.

Caddy es un reverse proxy con HTTPS automático (Let's Encrypt). Este Caddyfile contiene un bloque por cada tenant existente, escuchando en <tenant>.{{ cfg.base_domain }} y redirigiendo al puerto del Uptime Kuma correspondiente.

⬇ Descargar Caddyfile
📖 Pasos de instalación en el VPS
  1. En tu DNS (Cloudflare, NameCheap, etc.) creá un registro A wildcard: 
    *.{{ cfg.base_domain }}    A    {{ public_ip or 'IP_DEL_VPS' }}
    Si usás Cloudflare en modo proxy (nube naranja), poné nube gris para que Caddy maneje TLS directo.
  2. Conectate al VPS por SSH y instalá Caddy: 
    apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt update && apt install -y caddy
  3. Descargá el Caddyfile (botón arriba) y subilo al VPS: 
    scp Caddyfile root@{{ public_ip or 'IP_VPS' }}:/etc/caddy/Caddyfile
  4. Reload Caddy para aplicar: 
    systemctl reload caddy
    Caddy va a pedir certs Let's Encrypt automáticamente al primer request a cada subdominio.
  5. Cada vez que crees o borres un tenant, repetí los pasos 3 y 4 (o usá rsync/script para automatizarlo).
